# P2C Async Query Engine
This repository is based on the High Performance Query Processing SS25 Project by Lukas Neef & the Plan To Code (P2C) Query Engine by the DIS Chair at TUM. It contains the Async P2C Engine, which relies on the asynchronous scheduling principles described in the [AnyBlob paper](https://www.durner.dev/app/media/papers/anyblob-vldb23.pdf) to efficiently run compiled SQL queries on remotely stored [Parquet](https://parquet.apache.org/) files. The engine is built mainly for [TPCH](https://docs.aws.amazon.com/AmazonS3/latest/userguide/mountpoint.html) queries, but it can be adapted to run on arbitrary SQL tables.
A comprehensive description of the functionality and development process of the asynchronous P2C engine is available under `Docs/P2C_Thesis.pdf`.

## Setup
### Dependencies
A C++20-supporting compiler is necessary. Before running the engine with any compiled queries, a few dependencies must be installed:
- [OpenSSL](https://github.com/openssl/openssl)
- [LLVM](https://llvm.org/) (version 21 or higher)
- [jemalloc](https://github.com/jemalloc/jemalloc)
- [liburing](https://github.com/axboe/liburing)
- [AnyBlob](github.com/durner/AnyBlob/)
- [Apache Arrow/Parquet](https://arrow.apache.org/)
- [MoodyCamel ConcurrentQueue](https://github.com/cameron314/concurrentqueue)

**Note on Paths:** By default, the build looks for AnyBlob in `~/opt/anyblob-install` and ConcurrentQueue in `~/opt/concurrentqueue`. You can override these by setting the `ANYBLOB_ROOT` and `CONCURRENTQUEUE_ROOT` environment variables. Please take a look at `lib/CMakeLists.txt` for more details.

### Data Generation
The classic TPCH data generation suite, `tpch-dbgen`, was adapted to our custom Parquet format. For the purpose of generating the benchmark data and storing it in S3, the `data-generator/generate-data.sh` script can be used together with [S3-mount](https://docs.aws.amazon.com/AmazonS3/latest/userguide/mountpoint.html). Scale factor and chunk size can be specified.

### Config
A `config.json` file is necessary for the engine to run. Please rename the `config_example.json` to `config.json` (or copy it) and specify the cloud provider credentials. Furthermore, the remote bucket URI and the path to the parquet dataset must be specified. Other engine parameters may be tweaked, please refer to `Docs/` for a more comprehensive guide.

## How to use
Create a directory called `build`, inside of it run `cmake ..`. To compile individual queries, run `make tpch-q<QUERY_NR>`. Please note that the query plan is generated by [Umbra](https://umbra-db.com/). To run the actual query, run `./tpch-q<QUERY_NR>`. The `runs` environment variable determines how many times the query should run, and `use_async` determines if the asynchronous (optimized) mode should be used.
Example: `runs=5 use_async=1 ./tpch-q05`

## Benchmarking
A simple benchmark script, reliant on grid search for the scheduler parameters, is available at `p2c_bench.py`. To run the benchmarks, please create a Python venv using `python -m venv <venv_name>` and run `pip install -r requirements.txt`. Ensure [perf](https://perfwiki.github.io/main/) and [FlameGraph](https://perfwiki.github.io/main/) are installed and the correct paths are specified in the script. Afterwards, specify the desired parameter range/lists in `config_list.json` and simply run `python p2c_bench.py`. Please note that the affected parameters in `config.json` will be overwritten.
